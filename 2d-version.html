Hardwired CU Architecture
What is a Hardwired CU?
A Hardwired Control Unit generates control signals using fixed digital logic circuits — combinational logic gates, flip-flops, decoders, and state machines — instead of microprogram memory. It is blazingly fast but inflexible once designed.
Key Components
Instruction Register (IR) — holds the fetched instruction
Instruction Decoder — decodes opcode bits
Sequence Counter — tracks execution step T0–T5
Condition Logic — reads flags (Z,C,N,V)
Logic Gates Array — generates 32 control signals
Clock — synchronizes state transitions
Control Signal Generation
Each control signal Cₙ is a Boolean function:

Cₙ = f(IR, Tₛ, Flags)

Where IR is the opcode, Tₛ is timing step, and Flags are condition codes.
HCU vs Microprogrammed CU
Speed: HCU is 3–10× faster
Flexibility: Microprogrammed wins
Design Cost: HCU is harder to design
Area: HCU uses less silicon
Used in: RISC processors (ARM, MIPS)

Signal Timing Diagram · Control Signals vs Clock Cycles
Reading the Timing Diagram
Each row represents a control signal. A HIGH (1) pulse means the signal is active in that cycle. Signals are synchronized to the rising edge of the clock. The timing diagram directly corresponds to the truth table of the hardwired logic.
Cycle-by-Cycle Breakdown
T₀: Fetch — PC→MAR, read memory
T₁: Fetch — MDR→IR, PC++
T₂: Decode — opcode decoded
T₃: Execute — ALU operates
T₄: Memory access (LW/SW)
T₅: Writeback — result to Rd

How Hardwired Logic Uses This Table
Each cell in this table becomes a product term in a Sum-of-Products (SOP) Boolean expression implemented using AND/OR/NOT gates. The decoder expands the opcode bits, the counter selects the row (Tₛ), and the AND-array drives each column. This is essentially a Programmable Logic Array (PLA) structure.

Control Unit — Core Concepts
Definition
The Control Unit (CU) is the brain of the CPU. It orchestrates all operations by issuing control signals that direct the datapath components (ALU, registers, memory) when and how to operate.
Hardwired Design Philosophy
In a hardwired CU, the control logic is implemented directly in hardware as a finite state machine (FSM). The state transitions and output signals are fixed at design time. The FSM accepts:

• Opcode bits from IR
• Timing signals from sequence counter
• Status flags from ALU/registers
Finite State Machine Model
The HCU operates as a Moore FSM where outputs depend only on current state, or a Mealy FSM where outputs depend on state + inputs. Modern CPUs use Mealy-type for conditional branches.
Design Steps
1. Define instruction set & RTL operations
2. Map each operation to control signals
3. Build truth table (instruction × time step)
4. Minimize Boolean expressions (K-map / Quine-McCluskey)
5. Implement with gates or PLA
